# Telegram群组运营完全指南：从创建、管理到活跃社群的实战手册

Telegram 群组（Group）是 Telegram 最核心的社交功能之一。无论是兴趣交流、粉丝社群、项目社区还是工作协作，群组都是必不可少的工具。

但建群容易运营难。怎么建群、怎么设权限、怎么防广告、怎么提升活跃度？本文将从 0 到 1 带你完整掌握 Telegram 群组运营的全流程。

---

## 一、群组创建：从零开始建群

### 1.1 群组 vs 频道 vs 私聊：我该选哪个？

| 对比项 | 群组 (Group) | 频道 (Channel) | 私聊 (Chat) |
|:---|:---|:---|:---|
| **消息方向** | 多人双向聊天 | 单向广播 | 一对一 |
| **人数上限** | 超级群 20 万人 | 无上限 | 2 人 |
| **管理员标识** | 消息显示发送者 | 消息显示频道名 | 显示对方 |
| **适用场景** | 社群交流、讨论 | 内容发布、通知 | 私密对话 |
| **功能丰富度** | 最高（话题、语音、直播等） | 中等 | 基础 |

::: tip 选择建议
- 想**大家一起聊天讨论** → 选**群组**
- 想**发布内容通知** → 选**频道**
- 频道 + 群组搭配 → 频道发内容，群组做讨论（最佳实践）
:::

### 1.2 创建群组的具体步骤

**「新建群组」入口在哪里？**

- **iOS 客户端**：`对话` → `右上角最右边一个图标` → `新建群组`
- **Android 客户端**：`左上角三短线` → `新建群组`
- **Desktop 客户端**：`左上角三短线` → `新建群组`
- **macOS 客户端**：`主界面搜索框后面图标` → `新建群组`

::: details 界面截图

![tfa-menu.jpg](https://cdn.jsdelivr.net/gh/tgwiki/images/tfa/menu.jpg)

![td--menu.jpg](https://cdn.jsdelivr.net/gh/tgwiki/images/td/menu.jpg)

:::

**创建步骤：**

1. 点击「新建群组」
2. 选择要添加的成员（至少 1 人，可以先加个机器人）
3. 设置群组名称和头像
4. 创建完成！

::: tip 小技巧
创建新群时可以先拉一个机器人（比如 [Miss Rose](topics/bot/rose.md)）进去，
这样群创建好后机器人也在里面了，方便后续设置。
:::

### 1.3 普通群 vs 超级群 vs 广播群

Telegram 群组有三种形态：

| 类型 | 人数上限 | 特点 |
|:---|:---|:---|
| **普通群** | 200 人 | 基础功能，适合小圈子 |
| **超级群** | 20 万人 | 完整功能，公开搜索，适合社区 |
| **广播群** | 无上限 | 只有管理员能说话，成员可听语音 |

**普通群 → 超级群：**
- 当群人数达到一定数量（通常约 200 人），会自动升级为超级群
- 也可以手动升级：群设置 → 群组类型 → 设为公开

**超级群 → 广播群：**
- 超过 199,000 人可以转换为广播群组
- 转换后人数无限制，但只有管理员可以说话
- 成员仍然可以加入语音聊天

::: warning 注意
切换为广播群组之后，**不能回退**到普通群组。请谨慎操作！
:::

### 1.4 公开群 vs 私有群

| 类型 | 公开群组 (Public) | 私有群组 (Private) |
|:---|:---|:---|
| **搜索可见** | ✅ 可以通过搜索找到 | ❌ 搜索不到 |
| **加入方式** | 任何人可直接加入 | 需要邀请链接 |
| **历史记录** | 对所有人可见 | 新成员可设置是否可见 |
| **适合场景** | 公开社区、兴趣群 | 内部群、付费群、VIP群 |

**设置路径：** 群设置 → 群组类型 (Group Type)

::: tip 运营建议
- 刚起步的社群建议先**私有群**，质量可控
- 积累一定活跃度后再转**公开群**，方便引流
- 付费/ VIP 群一定要用**私有群** + 入群审核
:::

## 二、群组设置与权限管理

### 2.1 群组基础设置

**群设置入口：** 点击群名 → 点击「编辑」或「管理」

| 设置项 | 说明 | 建议 |
|:---|:---|:---|
| **群名称** | 群组的名字 | 简洁好记，包含关键词 |
| **群头像** | 群的 Logo/图标 | 清晰专业，和定位匹配 |
| **简介 (Bio)** | 群介绍，255 字以内 | 说明群是做什么的、群规 |
| **群组类型** | 公开/私有 | 根据运营阶段选择 |
| **慢模式** | 限制每人发言频率 | 广告多时可以开 |

### 2.2 成员权限设置

管理员可以限制所有成员或单个成员的权限。

**全局权限设置：** 群设置 → 权限 (Permissions)

| 权限项 | 说明 |
|:---|:---|
| **发送消息** | 能不能说话 |
| **发送媒体** | 能不能发图片、视频、文件 |
| **发送贴纸/GIF** | 能不能发表情包、动图 |
| **发送链接** | 能不能发链接（防广告神器） |
| **转发消息** | 能不能转发其他地方的消息 |
| **投票** | 能不能发起投票 |
| **添加成员** | 能不能拉人进群 |
| **置顶消息** | 能不能置顶消息（公开群普通成员不可用） |
| **修改群组信息** | 能不能改群名、头像等（公开群普通成员不可用） |

::: tip 防广告小技巧
开启「禁止发送链接」和「禁止转发消息」，可以挡住 80% 的广告党。
真有需要发链接的，可以让管理员代发。
:::

**个人权限设置：**
- 群设置 → 成员 → 点击某个人 → 权限
- 可以单独禁言某个人，或者限制其特定权限
- 也可以直接禁言/封禁

### 2.3 慢模式 (Slow Mode)

慢模式可以限制成员的发言频率，让讨论更有序。

**可选间隔：** 10秒 / 30秒 / 1分钟 / 5分钟 / 15分钟 / 1小时

**适用场景：**
- 讨论太激烈，消息刷太快
- 知识问答类群，让大家思考后再发言
- 活动进行时，控制节奏

::: tip 详细教程
慢模式完整介绍 → 参考 [慢速模式教程](./slowmode.md)
:::

### 2.4 入群审核

私有群和公开群都可以开启入群审核。

**开启后：**
- 用户申请入群需要管理员同意
- 管理员可以看到申请人的简介和头像
- 可以选择通过或拒绝

**设置路径：**
- 私有群：每个邀请链接可单独设置是否开启审核
- 公开群：群组类型里开启「入群审核」

::: tip 运营技巧
- 高质量社群一定要开入群审核
- 可以在群简介里说明加群要求（如"请备注来自哪里"）
- 过滤广告号和僵尸号
:::

## 三、管理员体系

### 3.1 添加管理员

群成员太多，一个人管不过来？添加管理员帮你一起管。

**添加步骤：**
1. 群设置 → 管理员 (Administrators) → 添加管理员
2. 选择要提升的成员
3. 勾选要授予的权限
4. 确认添加

### 3.2 管理员权限细分

Telegram 的管理员权限非常细，可以按需分配：

| 权限 | 说明 |
|:---|:---|
| **修改群组信息** | 改群名、头像、简介 |
| **删除消息** | 删除其他人的消息 |
| **封禁成员** | 踢人/封人 |
| **邀请成员** | 拉人进群 |
| **添加管理员** | 增加新的管理员 |
| **置顶消息** | 置顶/取消置顶消息 |
| **管理语音聊天** | 开启/管理语音通话 |
| **匿名发布** | 以群的名义发消息，不显示管理员名字 |

::: tip 权限分配原则
- **信任的核心管理员**：给全权限
- **普通管理（比如管广告的）**：只给删消息、封人权限
- **内容运营（发公告的）**：给置顶、发消息权限
- **不要随便给「添加管理员」权限**，这是最高权限之一
:::

### 3.3 自定义管理员头衔

可以给群主和管理员设置自定义头衔（比如「群主」「管理员」「客服」「技术支持」等）。

**设置方法：**
- 群设置 → 管理员 → 点击某个管理员 → 自定义头衔
- 头衔会显示在消息用户名旁边

### 3.4 近期操作日志

当多个管理员一起工作时，谁做了什么操作一目了然。

**查看路径：** 群设置 → 管理员 → 近期操作 (Recent Actions)

**记录内容：**
- 谁踢了谁
- 谁改了群设置
- 谁发了置顶消息
- 管理机器人的操作也会记录
- 保留过去 48 小时的记录

## 四、群组高级功能

### 4.1 话题 (Topics)

开启话题功能后，群组可以分成多个讨论区，类似 Discord 的频道。

**开启方式：** 群设置 → 话题 (Topics) → 开启

**特点：**
- 可以创建多个话题，每个话题有独立的聊天记录
- 每个话题可以设置图标和名字
- 话题有独立的链接，可以直接跳转
- General（综合讨论）话题可以隐藏

::: warning 注意
有关联频道（作为评论功能）的群组，**无法开启话题功能**。
:::

**适合场景：**
- 大型社区，讨论内容杂
- 项目社群，分不同模块讨论
- 学习群，分不同科目/章节

### 4.2 邀请链接管理

可以创建多个邀请链接，每个链接有不同的设置。

**设置路径：** 群设置 → 邀请链接 (Invite Links)

**每个链接可以设置：**
- 有效时间（多久过期）
- 使用次数限制（最多多少人用）
- 是否需要入群审核
- 链接名称（方便管理）

**运营用法：**
- 不同渠道用不同的链接，统计各渠道引流效果
- 活动用临时链接，活动结束就失效
- VIP 用专属链接，控制入群质量

### 4.3 语音聊天与直播

Telegram 群组支持语音聊天，升级后还能视频直播。

**语音聊天：**
- 任何群都可以发起
- 类似微信群语音，但人数上限更高
- 可以举手发言、静音管理

**视频直播：**
- 群组升级后可以开启视频直播
- 支持屏幕共享
- 适合线上分享会、讲座

### 4.4 投票 (Poll)

群里可以发起投票，增加互动。

**投票类型：**
- 单选投票
- 多选投票
- 匿名投票（看不到谁投了什么）
- 测验模式（有正确答案）

**运营用法：**
- 征集意见（下次活动做什么？）
- 知识测验（增加趣味性）
- 产品调研（用户想要什么功能？）

### 4.5 隐藏成员列表

开启后，非管理员无法查看完整的成员列表，只能看到管理员。

**适合场景：**
- 隐私要求高的群
- 防止其他群管理员来挖人
- 防止广告号批量加人

## 五、防广告与社群治理

### 5.1 常见的广告/骚扰类型

| 类型 | 表现 | 应对方法 |
|:---|:---|:---|
| **加群就发广告** | 进群发一条广告就跑 | 入群审核 + 新号限制 |
| **私聊广告** | 加群后私聊成员发广告 | 隐藏成员列表 + 提醒 |
| **换头像昵称冒充** | 冒充管理员骗人 | 设置管理员标识 + 群公告提醒 |
| **刷屏/炸群** | 一次性发大量消息 | 慢模式 + 删消息机器人 |
| **链接广告** | 发推广链接、邀请链接 | 禁止发链接权限 |
| **色情/垃圾信息** | 发不良内容 | 关键词过滤 + AI 反垃圾 |

### 5.2 官方反垃圾功能

Telegram 有内置的「强力反垃圾」功能。

::: caution 注意
该功能尚不完善，**误杀概率很大**，且被限制的用户无法在公开群组中发消息。
**不建议开启！** 推荐使用第三方机器人（如 Rose、Combot CAS）来替代。
:::

### 5.3 机器人辅助管理

这是最推荐的方式，功能强大且灵活。

| 机器人 | 核心功能 | 推荐度 |
|:---|:---|:---:|
| **Miss Rose** | 全能群管：欢迎语、过滤、踢人、禁言、锁定 | ⭐⭐⭐⭐⭐ |
| **Combot** | CAS 反垃圾 + 活跃度统计 + 积分排行 | ⭐⭐⭐⭐⭐ |
| **Shieldy** | 专门防机器人广告号进群 | ⭐⭐⭐⭐ |
| **Group Help** | 俄罗斯常用的群管机器人 | ⭐⭐⭐ |

::: tip 最佳实践
推荐 **Rose + Combot** 组合使用：
- Rose 负责日常管理（欢迎语、踢人、禁言、过滤）
- Combot 负责 CAS 反垃圾 + 数据统计 + 积分排行
:::

详细教程：
- [Miss Rose 机器人教程](topics/bot/rose.md)
- [Combot 机器人教程](topics/bot/combot.md)

### 5.4 群规制定

一个好的群规能减少 80% 的管理成本。

**群规建议包含：**

1. **禁止事项**：广告、刷屏、人身攻击、色情等
2. **违规处理**：警告 → 禁言 → 踢群
3. **加群要求**：比如"改备注"、"禁止小号"
4. **社群价值观**：提倡什么、鼓励什么

**群规放哪里？**
- 写在群简介里
- 发一条置顶消息
- 用机器人设置欢迎语，新人进群自动发送

## 六、活跃度运营

### 6.1 为什么群会死？

**常见死因：**
1. 没人说话，渐渐冷清
2. 广告太多，老用户都退了
3. 管理太严，没人敢说话
4. 没有持续的话题和价值输出

### 6.2 提升活跃度的方法

#### 方法一：定期话题/活动
- 每周固定一个主题讨论日
- 定期举办分享会、问答活动
- 节日活动、抽奖活动

#### 方法二：用排行榜激励
- 用 [Combot](topics/bot/combot.md) 的积分系统
- 每月评选活跃用户，给奖励
- 排行榜激发竞争欲

#### 方法三：内容/价值输出
- 定期分享干货、资源
- 邀请嘉宾做分享
- 群是提供价值的地方，不只是聊天

#### 方法四：营造社群氛围
- 管理员带头说话，不要只当删广告的
- 记住活跃用户的名字，增加亲切感
- 鼓励互帮互助，形成正向循环

### 6.3 群组数据分析

成员超过 500 人后，自动开通群组分析（统计数据）功能。

**可以查看：**
- 每日消息数趋势
- 活跃成员数
- 成员增长/流失
- 消息类型分布

**用数据优化运营：**
- 找到最活跃的时间段，在那个时间发公告
- 看成员流失和什么内容相关
- 追踪活动效果

## 八、常见问题 FAQ

### Q1: 群最多能有多少人？

普通群约 200 人，超级群 20 万人，广播群无上限。

### Q2: 可以转让群主吗？

可以。在管理员设置里，可以将群主身份转让给另一个管理员。
但注意：转让后你就不再是群主了，要谨慎操作。

### Q3: 删了群还能恢复吗？

删除群组后有 **一定的反悔期**（大约几小时内），期间可以撤销删除。
过了时间就真的没了，所以删群前要三思。

### Q4: 怎么看谁是僵尸号/广告号？

- 账号很新（注册时间短）
- 没有头像或头像很奇怪
- 用户名是随机字母数字组合
- 用 [Combot CAS](topics/bot/combot.md) 自动检测

### Q5: 成员列表里的"已删除账户"是什么？

是那些已经注销了 Telegram 账号的用户。
可以手动清理，也可以用机器人定期清理。

### Q6: 怎么把频道的评论区和群组合并？

在频道设置 → 讨论 → 添加讨论群组。
设置后，频道每条消息下方会出现"评论"按钮，点击进入关联的群组讨论。

## 九、总结：群组运营心法

**1. 定位清晰** — 知道群是做什么的，吸引什么人
**2. 规则先行** — 先定好规矩，后面少很多麻烦
**3. 价值为本** — 群要有价值，大家才愿意留
**4. 适度管理** — 太松乱，太紧死，找到平衡点
**5. 工具辅助** — 用好机器人，节省 80% 管理时间
**6. 持续运营** — 社群不是建完就完事了，需要持续投入
