# Miss Rose 机器人高级教程：规则系统、自动验证、黑名单与联邦封禁详解

如果你已经学会了 Miss Rose 的基础操作（欢迎语、过滤关键词、踢人禁言），想要解锁更多高级玩法，这篇文章就是为你准备的。

本文将深入讲解 Rose 的核心高级功能：规则系统、进群验证、联邦封禁、警告系统、定时消息等，帮你从群管小白进阶为群管专家。

## 一、警告系统 (Warnings)

Rose 有一个完整的警告系统，可以对违规用户进行梯度处罚。

### 1.1 为什么需要警告系统？

**传统方式的问题：**
- 一上来就踢人，太严厉，容易误伤
- 只靠管理员口头警告，没记录，用户不当回事

**警告系统的好处：**
- 梯度处罚：警告 1 次 → 警告 2 次 → 禁言 → 封禁
- 有记录可查，谁违规了、违规几次，一目了然
- 自动执行，不用管理员手动操作

### 1.2 警告相关指令

| 指令 | 功能 |
|:---|:---|
| `/warn` | 警告回复的用户（增加 1 次警告） |
| `/warnings` | 查看用户有多少次警告 |
| `/resetwarn` | 重置用户的警告次数 |
| `/warns` | 查看当前群组的警告设置 |

### 1.3 设置警告处罚

你可以自定义警告次数到达多少时，执行什么处罚。

**设置方法：**

```
/warnlimit 次数 处罚方式
```

**处罚方式有：**
- `ban` — 封禁
- `mute` — 禁言
- `kick` — 踢出
- `tban 时间` — 临时封禁（如 tban 1d = 封禁1天）
- `tmute 时间` — 临时禁言（如 tmute 2h = 禁言2小时）

**例子：**

```
/warnlimit 3 ban
```
→ 警告满 3 次自动封禁

```
/warnlimit 2 tmute 24h
```
→ 警告满 2 次禁言 24 小时

**时间格式：**
- `m` = 分钟（如 30m）
- `h` = 小时（如 12h）
- `d` = 天（如 7d）
- `w` = 周（如 2w）

## 二、规则系统 (Rules)

Rose 的规则系统非常强大，可以创建多个规则，每个规则对应不同的触发条件和处罚。

### 2.1 什么是规则？

规则就是：**当用户做了 X，就执行 Y**。

比如：
- 当用户发了"加群" → 自动回复群链接
- 当用户发了"微信" → 自动删除消息 + 警告
- 当用户发了链接 → 自动删除 + 禁言

### 2.2 添加规则

**基础格式：**

```
/addfilter 触发词 回复内容
```

这个是基础的过滤器（Filter），我们在基础教程里讲过了。

但高级玩法是：**不仅能回复，还能执行处罚动作**。

### 2.3 带动作的过滤器

使用 `{action}` 变量来指定触发后要做什么：

**格式：**

```
/addfilter 触发词
回复内容
{action: 动作}
```

**可用动作：**
- `ban` — 封禁用户
- `kick` — 踢出用户
- `mute` — 禁言用户
- `warn` — 警告用户
- `delete` — 只删除消息（不处罚）
- `nothing` — 什么都不做（只回复）

**例子 1：发广告直接踢**

```
/addfilter 加我微信
广告已删除，请不要发广告。
{action: kick}
```

**例子 2：发链接警告一次**

```
/addfilter http
请不要发链接，再发将被禁言。
{action: warn}
```

**例子 3：敏感词直接禁言 24 小时**

```
/addfilter 代充
请勿发布违规内容。
{action: tmute 24h}
```

### 2.4 正则表达式匹配（进阶）

Rose 支持正则表达式（Regex），可以实现更复杂的匹配。

**开启正则匹配：** 用 `/filter` 指令的模式匹配

**常用正则语法：**

| 语法 | 含义 | 例子 |
|:---|:---|:---|
| `^` | 匹配开头 | `^广告` = 以"广告"开头 |
| `$` | 匹配结尾 | `广告$` = 以"广告"结尾 |
| `.` | 任意一个字符 | `a.c` = aXc（X是任意字符） |
| `*` | 前面的字符出现0次或多次 | `aa*` = a, aa, aaa... |
| `+` | 前面的字符出现1次或多次 | `aa+` = aa, aaa, aaaa... |
| `\|` | 或 | `微信\|QQ` = 微信 或 QQ |
| `[]` | 字符集 | `[abc]` = a 或 b 或 c |

::: tip 提示
正则表达式是一个很强大的工具，但也比较复杂。
如果不会用，普通的关键词匹配也够用了。
:::

## 三、进群验证 (Captcha / Verification)

这是防机器人广告号最有效的功能之一。新成员进群后，必须通过验证才能说话。

### 3.1 为什么需要进群验证？

**广告机器人的套路：**
- 批量注册账号
- 自动加群
- 进群就发广告
- 管理员来不及删就已经被很多人看到了

**进群验证的作用：**
- 新成员进群后自动禁言
- 需要手动点击按钮/输入验证码才能解锁
- 机器人（脚本）很难通过验证
- 真人用户点一下就好，几乎没有门槛

### 3.2 开启进群验证

**开启按钮验证（最简单）：**

```
/captcha buttons
```

新成员进群后，会看到一个按钮，点击就能通过验证。

**其他验证方式：**

```
/captcha text     — 文字验证码（需要输入正确的文字）
/captcha math     — 数学题验证（比如 3+5=?）
/captcha off      — 关闭验证
```

**推荐：按钮验证 (Buttons)**
- 对真人最友好，点一下就行
- 但对简单的脚本也有一定防御效果
- 体验和安全的平衡点

### 3.3 设置验证时间限制

你可以设置必须在多长时间内完成验证，否则踢出：

```
/captcha kick 时间
```

**例子：**

```
/captcha kick 5m
```
→ 5 分钟内没通过验证就踢出

**时间格式：**
- `m` = 分钟
- `h` = 小时
- `d` = 天

### 3.4 验证相关指令

| 指令 | 功能 |
|:---|:---|
| `/captcha` | 查看当前验证设置 |
| `/captcha buttons` | 开启按钮验证 |
| `/captcha text` | 开启文字验证 |
| `/captcha math` | 开启数学题验证 |
| `/captcha off` | 关闭验证 |
| `/captcha kick 时间` | 设置超时踢出时间 |
| `/approve` | 手动通过某用户的验证 |

::: tip 最佳实践
对于公开群，强烈建议开启按钮验证。
虽然不能 100% 防住机器人，但能挡住 90% 以上的垃圾广告号。
:::

## 四、黑名单与白名单

### 4.1 关键词黑名单

就是我们前面讲的过滤器（Filter），但可以分不同的严重等级。

**推荐的分级策略：**

| 等级 | 内容 | 处罚 |
|:---|:---|:---|
| **轻度** | 广告关键词（如"加群"、"加我"） | 警告 1 次 |
| **中度** | 明显广告（如"代充"、"刷单"） | 禁言 24 小时 |
| **重度** | 色情、诈骗、违法内容 | 直接封禁 |

### 4.2 URL / 链接黑名单

不想让群里发链接？可以直接屏蔽所有链接。

**禁止所有链接：**

```
/lock url
```

**但这样太严格了？可以用黑名单/白名单方式：**

```
/addblacklistkeyword 域名
```

比如想屏蔽某竞品网站：

```
/addblacklistkeyword xxx.com
```

### 4.3 用户白名单

有些信任的用户（比如管理员、老成员），不想让他们被规则限制？

**添加白名单用户：**

```
/addtrust @用户名
```

**移除白名单：**

```
/removetrust @用户名
```

**查看白名单：**

```
/trustlist
```

白名单用户不会被过滤器、验证等规则限制。

## 五、联邦封禁 (Fed Bans)

这是 Rose 最强大的功能之一：多个群组联合起来，共享封禁名单。

### 5.1 什么是联邦封禁？

简单说就是：**一群群主联合起来，你封一个人，大家都跟着封。**

**举个例子：**
- A、B、C 三个群组成了一个"联邦"
- 有个人在 A 群发广告，被 A 群封禁了
- 这个人在 B 群和 C 群也会被自动封禁
- 一个广告号在一个群发广告，所有联邦群都会把它拒之门外

### 5.2 Rose 官方联邦

Rose 有一个官方维护的联邦，里面有很多大群。

**加入 Rose 官方联邦：**

```
/fedjoin RoseLabs
```

这样，所有被 Rose 官方联邦封禁的账号，进你的群就会被自动踢掉。

::: tip 提示
官方联邦的封禁标准比较严格，一般都是确认的广告号和骗子号才会被封。
对于大多数群来说，加入官方联邦就足够了。
:::

### 5.3 创建自己的联邦

如果你有多个群，或者和朋友的群想联合封禁，可以自己创建联邦。

**创建联邦：**

```
/newfed 联邦名称
```

**邀请其他群加入你的联邦：**
1. 在你的群里创建联邦
2. 把联邦 ID 告诉其他群主
3. 其他群主在他们的群里执行 `/fedjoin 联邦ID`

**联邦相关指令：**

| 指令 | 功能 |
|:---|:---|
| `/newfed 名称` | 创建一个新联邦 |
| `/fedjoin 联邦ID` | 加入一个联邦 |
| `/fedleave` | 退出当前联邦 |
| `/fedinfo` | 查看当前联邦信息 |
| `/fedadmins` | 查看联邦管理员 |

### 5.4 联邦封禁使用

在联邦群里封禁一个人，所有联邦群都会同步封禁。

**联邦封禁：**
直接在群里 `/ban` 用户，如果群在联邦里，会自动同步到所有联邦群。

**联邦封禁查询：**
```
/fbancheck 用户ID
```
查看这个用户在哪些联邦里被封禁了。

## 六、定时消息 (Scheduler)

Rose 可以设置定时消息，到点自动发送。

### 6.1 定时消息的用途

- 每天早上发早安/今日新闻
- 每周固定时间发群规提醒
- 活动开始前自动发通知
- 定期发推广内容（注意频率，别太频繁）

### 6.2 添加定时消息

**基础格式：**

```
/addtimer 时间间隔 消息内容
```

**时间间隔格式：**
- `每日几点`：`every day 时间`（如 every day 08:00）
- `每周几几点`：`every 周几 时间`（如 every monday 10:00）
- `每隔多久`：`every X hours`（如 every 24 hours）

**例子 1：每天早上 8 点发早安**

```
/addtimer every day 08:00
☀️ 早安！新的一天开始了！
今天也要元气满满哦~
```

**例子 2：每周一中午 12 点发提醒**

```
/addtimer every monday 12:00
📢 每周提醒：
请大家遵守群规，文明交流~
```

**例子 3：每 6 小时发一次**

```
/addtimer every 6 hours
⏰ 定时提醒：
本群禁止发广告，违者踢群！
```

### 6.3 管理定时消息

| 指令 | 功能 |
|:---|:---|
| `/timers` | 查看所有定时消息列表 |
| `/addtimer` | 添加定时消息 |
| `/removetimer 编号` | 删除某条定时消息 |

## 七、笔记系统 (Notes/Notes)

笔记功能可以让管理员保存一些常用内容，需要的时候一键调用。

### 7.1 什么是笔记？

就是把常用的内容存起来，用一个关键词触发。

比如：
- 存群规 → 发 `#群规` 就自动回复
- 存常见问题 → 发 `#价格` 就自动回复
- 存联系方式 → 发 `#联系` 就自动回复

和过滤器有点像，但区别是：
- **过滤器**是检测到关键词就自动触发（任何人发都触发）
- **笔记**需要 `#` 开头才触发，是主动调用的

### 7.2 添加笔记

```
/save 笔记名 笔记内容
```

**例子：**

```
/save 群规
📋 本群群规：
1. 禁止广告
2. 禁止人身攻击
3. 禁止发色情内容
违者禁言/踢群！
```

使用时，在群里发：
```
#群规
```

Rose 就会自动回复保存的内容。

### 7.3 笔记相关指令

| 指令 | 功能 |
|:---|:---|
| `/save 名称 内容` | 保存一条笔记 |
| `/get 名称` | 获取笔记内容 |
| `/notes` | 查看所有笔记列表 |
| `/clear 名称` | 删除一条笔记 |

## 八、欢迎语高级技巧

### 8.1 欢迎语变量大全

在欢迎语中可以使用这些变量，Rose 会自动替换：

| 变量 | 替换为 |
|:---|:---|
| `{first}` | 用户的名字 |
| `{last}` | 用户的姓氏 |
| `{fullname}` | 用户的全名 |
| `{username}` | 用户的用户名（@xxx） |
| `{id}` | 用户的 ID |
| `{mention}` | @用户（点击可跳转） |
| `{chatname}` | 当前群的名字 |
| `{count}` | 当前群的人数 |

**高级欢迎语例子：**

```
/setwelcome
🎉 欢迎 {fullname} 加入{chatname}！
你是第 {count} 位成员~

📋 请先阅读置顶群规
❓ 有问题可以发 #常见问题
```

### 8.2 离开消息

也可以设置用户离开时的消息（可选）：

```
/setgoodbye {fullname} 离开了我们，再见~
```

**关闭离开消息：**

```
/setgoodbye no
```

## 九、权限锁定 (Locks)

Rose 可以精细化控制群成员的各种权限。

### 9.1 可以锁定的项目

| 指令 | 锁定内容 |
|:---|:---|
| `/lock all` | 锁定所有（全体禁言） |
| `/unlock all` | 解锁所有 |
| `/lock url` | 禁止发链接 |
| `/lock forward` | 禁止转发消息 |
| `/lock sticker` | 禁止发贴纸 |
| `/lock gif` | 禁止发 GIF 动图 |
| `/lock audio` | 禁止发语音 |
| `/lock photo` | 禁止发图片 |
| `/lock video` | 禁止发视频 |
| `/lock file` | 禁止发文件 |

**查看当前锁定设置：**

```
/locks
```

### 9.2 定时锁定

配合定时消息，可以实现定时禁言。

比如：晚上 12 点到早上 8 点禁言，防止夜猫子刷屏。

可以用定时消息 + 锁定指令来实现：
```
/addtimer every day 23:59
🌙 夜深了，大家早点休息~
群已开启禁言，明天早上 8 点解锁。
{command: lock all}
```

```
/addtimer every day 08:00
☀️ 早安！群已解锁~
开始新的一天吧！
{command: unlock all}
```

## 十一、避坑指南与常见问题

### Q1: 为什么机器人没有反应？

**排查：**
1. 机器人在群里吗？
2. 机器人是管理员吗？给全权限了吗？
3. 指令是不是在群里发的？（有些指令只能在群里用）
4. 是不是指令打错了？（注意斜杠 `/` 和拼写）

### Q2: 机器人删消息不及时怎么办？

- 确认机器人有「删除消息」权限
- 如果是链接/广告，可以用过滤器设置自动删除
- 开启进群验证，从源头减少广告号

### Q3: 规则太多会误杀吗？

会的。所以：
- 关键词别太宽泛（比如只设"微"可能会误杀很多）
- 轻度违规用警告，别一上来就踢
- 管理员多关注，有问题及时调整
- 白名单功能可以保护信任的用户

### Q4: 联邦封禁会误封吗？

官方联邦的审核比较严格，误封概率很低。
如果是自己建的小联邦，要注意管理员素质，避免滥用封禁权。

### Q5: 机器人是免费的吗？

Miss Rose 基础功能完全免费。
高级功能（如更多定时消息、联邦封禁数量上限等）可能需要捐赠，但大部分群用免费版就足够了。

## 十二、总结：进阶群管最佳实践

**1. 基础配置**
- 开启进群验证（按钮模式）
- 设置欢迎语 + 群规
- 设置关键词过滤器

**2. 安全配置**
- 加入 Rose 官方联邦（RoseLabs）
- 配合 Combot CAS 双重防护
- 设置梯度警告（警告 → 禁言 → 封禁）

**3. 运营配置**
- 设置定时提醒（群规、活动通知）
- 用笔记功能整理 FAQ
- 用排行榜激励活跃（配合 Combot）

**4. 团队协作**
- 合理分配管理员权限
- 用近期操作日志监督管理行为
- 定期复盘管理效果

::: tip 最后想说
机器人只是工具，真正让社群变好的是运营者的用心。
Rose 能帮你节省 80% 的重复劳动，但剩下的 20% —— 社群氛围、价值输出、用户关系，还是要靠你自己。
祝你运营出一个高质量的 Telegram 社群！🚀
:::
